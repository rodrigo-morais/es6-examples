EcmaScript6 Examples - modules - default

ES6 has a feature to export a single module called default. When the export uses the reserved word "default" is not necessary to use the same name of variable as the module's name. The destructuring is implicit, and the name is insignificant.

Let's see an example: 

                
                    //.employee.js
                    export class Employee {
                        constructor(name) {
                            this._name = name;
                        }

                        get name() {
                            return this._name;
                        }

                        doWork() {
                            return '${this._name} is working';
                        }
                    }

                    //.module_error.js
                    import factory from './employee.js'

                    var employee = new factory('Rodrigo');
                    employee.doWork(); //Rodrigo is working

In this case the export is not using the reserved word "default", and because of that is necessary that the variable which receives the module has the same name of the module. If it doesn't happen the code returns an error. 

                
                    //.employee_default.js
                    export default class Employee {
                        constructor(name) {
                            this._name = name;
                        }

                        get name() {
                            return this._name;
                        }

                        doWork() {
                            return '${this._name} is working';
                        }
                    }

                    //.module_default.js
                    import factory from './employee.js'

                    var employee = new factory('Rodrigo');
                    employee.doWork(); //Rodrigo is working

In this case the export is using the reserved word "default", and because of that is not necessary that the variable which receives the module has the same name of the module.
